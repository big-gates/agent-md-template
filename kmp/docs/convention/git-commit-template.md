# Git 커밋 템플릿

## 1) 제목 형식 (필수)

```
<type>: <summary>
```

- `type`은 아래 중 하나만 사용한다.
  - `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `build`, `ci`
- `summary`는 50자 내외로 변경의 핵심 결과만 작성한다.
- `summary`는 한국어로 작성한다(고유명사/코드 식별자는 원문 유지 가능).
- 제목 끝에 마침표를 붙이지 않는다.

예시:

- `docs: 에이전트 커밋/PR 템플릿 추가`
- `fix: trSeq 채번 중복 방지 로직 보완`

## 2) 본문 형식 (선택, 권장)

복잡한 변경은 본문을 반드시 추가한다.
본문의 문장도 한국어로 작성한다(명령어/코드/식별자는 원문 유지 가능).

```
<type>: <summary>

Why:
- 왜 이 변경이 필요한지

What:
- 무엇을 바꿨는지
- 영향 범위

Test:
- 실행한 검증 (`./gradlew test` 등)
- 미실행 시 이유
```

## 3) 에이전트 체크리스트

- 제목이 `<type>: <summary>` 형식인가?
- `type`이 허용 목록에 있는가?
- 요약이 결과 중심으로 간결한가?
- 제목 `summary`가 한국어인가?
- 복잡한 변경이면 `Why/What/Test`를 포함했는가?
- 본문 문장이 한국어인가?
