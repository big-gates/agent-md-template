# Pull Request Convention

## PR 제목

커밋 컨벤션과 동일한 형식을 따른다.

```
feat(auth): 로그인 폼 유효성 검사 추가
```

## PR 본문 템플릿

```markdown
## 변경 사항
- 어떤 변경을 했는지 요약

## 변경 이유
- 왜 이 변경이 필요한지 설명

## 스크린샷 (선택)
- UI 변경이 있는 경우 스크린샷 첨부

## 테스트
- [ ] 로컬에서 정상 동작 확인
- [ ] 관련 테스트 코드 추가/수정

## 리뷰 포인트 (선택)
- 리뷰어가 집중적으로 봐야 할 부분
```

## PR 규칙

| 규칙 | 설명 |
|---|---|
| 단일 관심사 | 하나의 PR은 하나의 관심사만 다룬다 |
| 분리 | 변경 파일이 많을 경우 기능 단위로 PR을 분리한다 |
| Draft PR | 진행 중인 작업을 공유할 때 활용한다 |
| 리뷰 요청 | 리뷰어 지정 후 Slack으로 알린다 |
